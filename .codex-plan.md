### Step 1 -- Replace all bare eval in grounding_agent_uitars.py

**File:** `src/khoj/processor/operator/grounding_agent_uitars.py`

**1A. Add a helper function** near the top of the file (after imports, before the class):

```python
def _parse_box_literal(raw: str) -> tuple[float, ...]:
    """Parse an LLM-produced coordinate literal safely."""
    value = ast.literal_eval(raw)
    if not isinstance(value, (list, tuple)) or len(value) not in (2, 4):
        raise ValueError(f"Invalid box literal (expected 2 or 4 elements): {raw!r}")
    return tuple(float(v) for v in value)
```

The helper must accept both 2-element `(x, y)` and 4-element `(x1, y1, x2, y2)` tuples because the click-action parser at line 720 handles both cases.

### Step 2 -- Remove `shell=True` from `docker_execute()`

**File:** `src/khoj/processor/operator/operator_environment_computer.py`

Replace lines 622-658 (the entire `docker_execute` method) with:

```python
async def docker_execute(self, python_command_str: str) -> Optional[str]:
    """Execute a Python command inside the Docker container.

    This and _execute() are the only allowed Python-code execution paths.
    """
    if not self.docker_container_name or not self.docker_display:
        logger.error("Container name or Docker display not set for Docker execution.")
        return None

    cmd = [
        "docker", "exec",
        "-e", f"DISPLAY={self.docker_display}",
        self.docker_container_name,
        "python3", "-c", python_command_str,
    ]

    try:
        process = await asyncio.to_thread(
            subprocess.run,
            cmd,
            shell=False,
            capture_output=True,
            text=True,
            check=False,
            timeout=120,
        )
        if process.returncode != 0:
            if "FailSafeException" in process.stderr or "FailSafeException" in process.stdout:
                raise KeyboardInterrupt(process.stderr or process.stdout)
            else:
                logger.error(
                    f"Docker command failed. Return Code: {process.returncode}. "
                    f"Stderr: {process.stderr[:500]}"
                )
                raise RuntimeError(f"Docker exec error: {process.stderr or process.stdout}")
        return process.stdout.strip()
    except KeyboardInterrupt:
        raise
    except Exception as e:
        logger.error(f"Unexpected error running command in Docker: {type(e).__name__}")
        raise RuntimeError(f"Unexpected Docker error: {e}") from e
```

Also add a clarifying comment to `_execute_shell_command()` around line 518:

```python
# Intentionally shell-backed: supports terminal commands, pipes,
# redirections, and Docker bash -c execution. Not a Python-code path.
```

**Closure condition:** `docker_execute()` uses an argv list with `shell=False`. The old `safe_python_cmd` string-quoting hack and the `docker_full_cmd` f-string are both gone.

## Acceptance gate

This plan is closed only when all of the following are true:

1. `grounding_agent_uitars.py` has zero remaining bare `eval(` calls.
2. `docker_execute()` uses an argv list with `shell=False`.
3. `DB_NAME` is validated against `^[A-Za-z_][A-Za-z0-9_]*$` before any SQL interpolation.
4. No `${{ }}` expressions appear inside `run:` shell bodies in `run_evals.yml`.
5. There is no runtime `!secret` fallback anywhere in the Django/Starlette secret path.
6. Default user password is generated with `secrets.token_urlsafe()`, not hardcoded.
7. CORS no longer uses wildcard methods/headers or invalid origin patterns.
8. Jinja email templates default to autoescape, with only intentionally rendered markdown HTML marked safe.
9. CSP does not include `'unsafe-eval'` (or it is documented as temporary tracked debt).
10. Both Dockerfiles run as non-root.
